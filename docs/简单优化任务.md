# 妙语成语 - 简单优化任务

这些任务改动范围小、依赖少，适合快速分给同事并行处理。建议每个任务单独提交，方便回滚。

## 1. 移除打包进 rawfile 的脚本

**背景：** 构建时提示 `entry/src/main/resources/rawfile/gen_chengyu.py` 被打进包里。

**目标：**
- 不把 Python 生成脚本打进应用包。

**建议改法：**
- 将 `entry/src/main/resources/rawfile/gen_chengyu.py` 移到 `scripts/` 或 `entry/src/main/resources/rawfile/tools/` 之外的目录。
- 如果脚本仍有用，在 README 或脚本目录中说明用途。

**涉及文件：**
- `entry/src/main/resources/rawfile/gen_chengyu.py`
- 可选：`README.md`

**验收：**
- `hvigorw assembleApp --no-daemon` 不再出现 source code files packaged warning。

## 2. 文档统一

**背景：** README、功能文档、测试报告对数据量、难度、提示逻辑描述不一致。

**目标：**
- 统一项目当前事实，减少后续维护误判。

**建议改法：**
- README 数据量统一为 510 条。
- 难度统一为 1-3。
- 分类统一为 10 类。
- 删除或归档历史通信故障文档里的过期描述。
- 更新 `TestReport.md` 中 50 条数据、旧提示逻辑等过期内容。

**涉及文件：**
- `README.md`
- `docs/功能文档.md`
- `TestReport.md`
- `PROJECT_STATUS.md`
- `COMPLETE_STATUS.md`

**验收：**
- 文档中不再出现互相矛盾的数据量、分类数、难度范围。

## 3. 路由 API 迁移预处理

**背景：** 构建 warning 显示 `pushUrl`、`replaceUrl`、`back`、`getParams` 已弃用。

**目标：**
- 先整理路由调用点，为后续统一迁移做准备。

**建议改法：**
- 保持页面不直接调用 `router`。
- 所有跳转继续只走 `NavUtil.ets`。
- 搜索页面中是否存在绕过 `NavUtil` 的路由调用，若有统一收敛。

**涉及文件：**
- `entry/src/main/ets/utils/NavUtil.ets`
- `entry/src/main/ets/pages/*.ets`

**验收：**
- `rg "router\\." entry/src/main/ets` 只在 `NavUtil.ets` 中命中。

## 4. AlertDialog 弹窗封装

**背景：** 构建 warning 显示 `AlertDialog.show` 已弃用。

**目标：**
- 先封装统一弹窗入口，减少后续 API 替换成本。

**建议改法：**
- 新增 `utils/DialogUtil.ets`。
- 提供 `showInfoDialog(title, message, confirmText)`。
- 替换 `GamePage.ets` 和 `CollectionPage.ets` 中直接调用 `AlertDialog.show` 的代码。

**涉及文件：**
- `entry/src/main/ets/utils/DialogUtil.ets`
- `entry/src/main/ets/pages/GamePage.ets`
- `entry/src/main/ets/pages/CollectionPage.ets`

**验收：**
- 页面不再直接调用 `AlertDialog.show`。
- 构建至少不新增 warning 或错误。

## 5. 收藏页首字母显示复查

**背景：** 当前已改为使用拼音字段提取首字母，但收藏页可以顺手补几个视觉细节。

**目标：**
- 图鉴列表首字母、拼音、释义显示稳定，不因长文本撑破布局。

**建议改法：**
- 首字母限制单行。
- 释义最多两行，超出省略。
- 弹窗详情中显示完整拼音、释义、出处。

**涉及文件：**
- `entry/src/main/ets/pages/CollectionPage.ets`

**验收：**
- 长释义成语列表项不溢出。
- 拼音带声调时首字母正确，例如 `èr` 显示 `E`。

## 6. 首页默认入口优化

**背景：** 用户每天打开时，最可能想继续玩或做每日挑战。

**目标：**
- 降低首次操作成本。

**建议改法：**
- 在首页游戏 Tab 顶部增加“今日挑战”或“继续挑战”入口。
- 保留现有模式、难度、分类选择。

**涉及文件：**
- `entry/src/main/ets/pages/HomePage.ets`

**验收：**
- 打开首页首屏能直接看到今日挑战入口。
- 不影响原计时模式、闯关模式入口。

## 7. 简单成就补齐

**背景：** 目前已有成就系统，但可以补几个和图鉴/每日挑战相关的基础成就。

**目标：**
- 增加轻量目标感。

**建议成就：**
- 第一次完成每日挑战。
- 连续 3 天完成每日挑战。
- 收藏 10 个成语。
- 收藏 50 个成语。

**涉及文件：**
- `entry/src/main/ets/model/AchievementModel.ets`
- `entry/src/main/ets/service/AchievementService.ets`
- `entry/src/main/ets/pages/DailyChallengePage.ets`
- `entry/src/main/ets/service/CollectionService.ets`

**验收：**
- 成就页能看到新增成就。
- 达成条件后能正确解锁或增加进度。
